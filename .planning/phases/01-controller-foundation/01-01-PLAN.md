**Controller — `lib/src/controller/ai_chat_scroll_controller.dart`:**

Implement exactly as specified in 01-RESEARCH.md "Controller — Full Phase 1 Implementation" section:
- Class `AiChatScrollController extends ChangeNotifier`
- Private `ScrollController? _scrollController` field
- `attach(ScrollController)` — assigns field, assert not already attached
- `detach()` — nulls field
- `onUserMessageSent()` — wraps logic in `SchedulerBinding.instance.addPostFrameCallback`, guards with `_scrollController == null || !_scrollController!.hasClients` null/hasClients check, calls `notifyListeners()`. Phase 1: the callback body is a no-op beyond the guard (actual jump logic is Phase 3).
- `onResponseComplete()` — calls `notifyListeners()`. Phase 1: no streaming state to clear yet.
- `dispose()` — nulls `_scrollController`, calls `super.dispose()`
- Full dartdoc on the class and every public method (per research examples)
- Import `package:flutter/scheduler.dart` for SchedulerBinding, `package:flutter/widgets.dart` for ScrollController/ChangeNotifier

**Widget stub — `lib/src/widgets/ai_chat_scroll_view.dart`:**

Implement exactly as specified in 01-RESEARCH.md "Widget — Phase 1 Stub" section:
- `AiChatScrollView extends StatefulWidget` with `required this.controller` (AiChatScrollController) and `required this.child` (Widget)
- `_AiChatScrollViewState`:
  - `late final ScrollController _scrollController`
  - `initState()`: creates ScrollController, calls `widget.controller.attach(_scrollController)`
  - `dispose()`: calls `widget.controller.detach()`, then `_scrollController.dispose()`, then `super.dispose()`
  - `build()`: returns `widget.child` (pass-through stub — Phase 2 replaces with CustomScrollView)
- Full dartdoc on the class and constructor params
