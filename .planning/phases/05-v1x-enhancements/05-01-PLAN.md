  <behavior>
    - Test 1: controller.isAtBottom is a ValueListenable&lt;bool&gt; and starts as true (no messages = at bottom)
    - Test 2: After user scrolls up (away from bottom), isAtBottom.value becomes false
    - Test 3: After user scrolls back down to bottom, isAtBottom.value becomes true
    - Test 4: scrollToBottom() animates scroll to maxScrollExtent and isAtBottom becomes true
    - Test 5: After onUserMessageSent() + anchor pipeline, isAtBottom is false (user is anchored at sent message, not at bottom during streaming)
    - Test 6: After onResponseComplete() when user has not scrolled, isAtBottom reflects actual scroll position
    - Test 7: isAtBottom uses a threshold (default 50.0 logical pixels) so near-bottom counts as "at bottom"
  </behavior>
  <action>
  **Controller changes (ai_chat_scroll_controller.dart):**

  1. Add a `ValueNotifier<bool> _isAtBottom = ValueNotifier(true)` field.
  2. Expose `ValueListenable<bool> get isAtBottom => _isAtBottom` (import `package:flutter/foundation.dart` for ValueListenable).
  3. Add an internal method `void updateIsAtBottom(bool value)` that sets `_isAtBottom.value = value` (called by the widget state).
  4. Add `void scrollToBottom()` method:
     - Guard: if `_scrollController == null || !_scrollController!.hasClients` return.
     - Call `_scrollController!.animateTo(_scrollController!.position.maxScrollExtent, duration: Duration(milliseconds: 300), curve: Curves.easeOut)`.
     - Note: the scroll listener in the widget will update isAtBottom as the animation progresses.
  5. Add `double atBottomThreshold = 50.0` as a constructor parameter (optional, named) with default 50.0.
  6. In `dispose()`, add `_isAtBottom.dispose()` before `super.dispose()`.

  **Widget changes (ai_chat_scroll_view.dart):**

  1. In `_onScrollChanged()`, after the existing filler recomputation logic, add isAtBottom tracking:
     ```dart
     // Always track isAtBottom, regardless of anchor state
     if (_scrollController.hasClients) {
       final pos = _scrollController.position;
       final atBottom = pos.maxScrollExtent - pos.pixels <= widget.controller.atBottomThreshold;
       widget.controller.updateIsAtBottom(atBottom);
     }
     ```
     IMPORTANT: This tracking must run even when `_anchorActive` is false. The existing `_onScrollChanged` early-returns when `!_anchorActive`. Restructure: extract the isAtBottom check to run unconditionally, keep the filler scheduling gated behind `_anchorActive`.

  2. Also listen to `ScrollMetricsNotification` for isAtBottom (already done via `_onScrollChanged` call in the notification listener).

  3. In `_onControllerChanged()` when streaming starts (anchor pipeline), do NOT set isAtBottom to false -- let the scroll listener handle it naturally based on actual position.

  **Do NOT add a widget parameter for threshold** -- it lives on the controller where devs configure it.
  </action>
